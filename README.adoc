= UDAL вҖ” Universal Device Abstraction Layer
:toc:
:toc-placement: left

image:https://img.shields.io/badge/status-pre--alpha-red[]
image:https://img.shields.io/badge/license-Apache%202.0-blue[]

UDAL provides a unified API surface for interacting with heterogeneous IoT devices regardless of their underlying transport protocol.
Applications talk to one consistent gRPC/REST interface; UDAL routes requests to the correct transport adapter.

== Architecture

The system is documented as code using link:https://github.com/docToolchain/Bausteinsicht[Bausteinsicht].
See link:architecture.jsonc[architecture.jsonc] for the full model with system context, gateway internals, SDK overview, and sequence diagrams.

== Non-Functional Requirements (v1)

|===
| Requirement | Target

| End-to-end latency (p99)
| < 50 ms

| Concurrent devices (single instance)
| 1,000

| Deployment targets
| Single binary, Docker, Kubernetes

| Transport protocols
| MQTT (3.1.1 + 5), HTTP, CAN (SocketCAN)

| Authentication
| API-Key, mTLS, OAuth2 Device Flow

| Authorisation
| RBAC + per-device ACL

| Persistence
| Device Registry (embedded bbolt)
|===
